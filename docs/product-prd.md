# JD-to-Demo Product PRD

## 1. 产品定位

JD-to-Demo 是一个面向低年级学生的 AI 求职项目工作台。

它帮助没有正式实习经历的学生，把目标岗位 JD 转换成一个可开发、可展示、可写进简历、可拿给 HR 沟通的 demo 方案。

一句话：

> 根据目标岗位要求，快速生成一个强相关的项目 demo brief，并配套开发计划、简历表达和 HR 沟通材料。

## 2. 用户画像

### 核心用户

- 大二/大三学生。
- 正在找 AI 应用、全栈工程师、前端、后端、产品技术类实习。
- 有课程项目或自学经历，但缺少正式实习。
- 不知道如何把岗位要求转成可以展示的作品。

### 用户当前困境

- 岗位要求经常写“有相关实习/项目经验优先”。
- 简历上的项目和岗位业务不够贴合。
- 做项目容易从技术出发，而不是从招聘方需求出发。
- 和 HR 沟通时缺少一个能快速证明能力的材料。
- 不知道项目应该做到多大，容易范围失控。

## 3. 核心价值

这个产品的核心价值不是替用户“编项目”，而是帮助用户建立一条可信的能力证明链：

```text
岗位 JD -> 能力信号 -> demo 场景 -> MVP 功能 -> 工程实现 -> 展示材料 -> HR 沟通
```

它要解决三个问题：

1. 做什么：根据岗位选择最相关的 demo。
2. 怎么做：把 demo 压缩成 1-3 天可交付的 MVP。
3. 怎么展示：把 demo 转换成简历 bullet、演示脚本和 HR 话术。

## 4. 第一版 MVP

第一版不做账号系统、不做复杂项目管理，只做一个高质量单页工作台。

### 输入

- 目标岗位名称。
- 公司/业务描述。
- 原始岗位 JD。
- 用户已有技术栈。
- 可投入时间。
- 用户已有优势。

### 处理

- 提取岗位关键词。
- 识别能力信号。
- 判断业务场景。
- 推荐 demo 类型。
- 生成 MVP 功能列表。
- 生成 24/48/72 小时构建计划。
- 生成简历 bullet。
- 生成 HR 开场话术。

### 输出

- 项目名称。
- 一句话定位。
- 岗位能力信号。
- 证明方式。
- MVP 功能。
- 技术栈建议。
- 构建计划。
- 简历 bullet。
- HR 话术。

## 5. 非目标

第一版暂时不做：

- 自动生成完整生产级代码。
- 自动投递岗位。
- 伪造实习经历。
- 复杂简历排版。
- 长周期项目管理。
- 多人协作。

原因：这些会让产品过早变重。第一版最重要的是证明“JD 到 demo brief”的转换足够有用。

## 6. 关键设计原则

### 强岗位相关

输出内容必须直接回应岗位 JD，而不是泛泛推荐“做一个 AI 助手”。

### 小而完整

demo 要控制在 1-3 天内能完成，但必须有真实输入、处理过程、结果输出和展示材料。

### 能被 HR 快速理解

HR 不一定懂技术细节，所以输出必须包含业务场景、项目定位和可读话术。

### 能被面试官追问

面试官会追问实现细节，所以输出必须包含技术栈、功能边界、数据结构或 AI prompt 方向。

### 不鼓励造假

系统帮助用户做真实 demo，不帮助用户包装不存在的经历。

## 7. 推荐信息架构

### 工作台

- 左侧：岗位信息输入。
- 右侧：方案输出。
- 顶部：复制、导出、示例按钮。

### 方案结果

- 项目定位。
- 能力信号。
- demo 功能。
- 技术方案。
- 开发计划。
- 展示材料。

### 案例库

- AI 应用开发实习生案例。
- 全栈工程师实习生案例。
- 前端工程师实习生案例。
- 后端/平台开发实习生案例。

## 8. 未来版本路线图

### V0.0 Codex Skill

- 先沉淀 `jd-to-demo-workflow` skill。
- 支持岗位截图/照片或 JD 文本分析。
- 支持结合个人背景、技术栈、时间限制和展示诉求。
- 输出完整 Codex 项目生成提示词、简历 bullet、HR 话术和演示脚本。

### V0.1 本地单页工具

- 本地 HTML 可打开。
- 基于规则识别 JD。
- 生成结构化 demo brief。
- 支持复制结果。

### V0.2 AI 生成增强

- 接入 LLM API。
- 支持更细的 JD 解析。
- 输出更具体的页面、接口、数据结构和 prompt。
- 支持用户选择 demo 类型。

### V0.3 项目脚手架

- 根据 demo brief 生成前端页面骨架。
- 生成 README。
- 生成 mock 数据。
- 生成 API 路由草稿。

### V0.4 展示材料中心

- 自动生成项目截图清单。
- 生成 90 秒演示脚本。
- 生成多版本简历 bullet。
- 生成不同语气的 HR 话术。

### V0.5 投递反馈闭环

- 记录投递岗位。
- 记录 HR 反馈。
- 分析哪些 demo 类型转化更好。
- 为相似岗位推荐复用方案。

## 9. 成功指标

### 对用户有用

- 用户粘贴一个 JD 后，10 分钟内能确定 demo 方向。
- 用户能在 48 小时内完成第一个可展示 demo。
- 输出内容可以直接改写进简历和 HR 消息。

### 对求职有效

- HR 回复率提升。
- 面试中项目被追问。
- 用户能讲清楚项目和岗位的对应关系。
- 同一个 demo 底座可以复用到多个相似岗位。

## 10. 展示给 HR 的项目叙述

可以这样介绍这个项目：

> 我注意到低年级学生找 AI/全栈实习时，经常缺少正式实习经历，也不知道如何把岗位要求转换成能展示的项目。因此我做了一个 JD-to-Demo 工作流工具：用户粘贴招聘 JD 后，系统会提取能力信号，推荐强相关 demo 方向，并生成 MVP 功能、开发计划、简历 bullet 和 HR 沟通话术。这个项目本身也体现了我对 AI 应用落地、全栈产品设计和真实求职场景的理解。
