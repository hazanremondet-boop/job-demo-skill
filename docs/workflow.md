# AI / Full-Stack Internship Demo Workflow

这套工作流的目标不是“随便做个项目”，而是把目标岗位的要求转换成一个可以快速交付、可以展示、可以写进简历、可以在面试里展开讲的 demo。

## 1. 岗位信息采集

把岗位 JD 复制到 `templates/job-intake.md`，重点记录：

- 公司/团队在做什么业务。
- 岗位职责里重复出现的动词，比如开发、优化、接入、搭建、部署、分析。
- 技术要求里的硬技能，比如 React、Vue、Node.js、Python、LLM、RAG、Agent、SQL、Docker。
- 隐含能力，比如产品意识、工程落地、数据处理、沟通协作、快速学习。
- 你当前已有的相关经历、课程作业、比赛、个人项目。

## 2. 把 JD 转成能力信号

每个要求都要翻译成“我可以用什么证据证明”。

| JD 要求 | 能力信号 | demo 证据 |
| --- | --- | --- |
| 熟悉 AI 应用开发 | 能把 LLM 接到真实业务流程 | 做一个有输入、推理、输出、反馈的 AI 工具 |
| 熟悉前后端开发 | 能完成完整产品闭环 | 前端页面 + 后端 API + 数据存储 |
| 熟悉 RAG | 能处理私有知识库问答 | 上传文档、检索片段、生成回答、引用来源 |
| 有产品意识 | 能把需求压缩成可用功能 | 明确用户、场景、痛点、核心指标 |
| 有部署经验 | 能让别人直接访问 | Vercel/Render/云服务器部署链接 |

## 3. 选择 demo 类型

优先选择贴近公司业务的“小工具”，不要做过于泛化的玩具项目。

### AI 应用岗位

- 面向客服团队：知识库问答助手、工单摘要助手、投诉分类助手。
- 面向运营团队：活动文案生成器、用户反馈聚类、竞品评论分析。
- 面向销售团队：客户线索评分、拜访纪要总结、销售话术生成。
- 面向招聘团队：简历筛选助手、候选人匹配评分、面试题生成器。
- 面向教育业务：错题讲解助手、课程内容问答、学习计划生成器。

### 全栈工程师岗位

- 后台管理系统：用户、权限、数据表、筛选、导出。
- 数据看板：指标卡片、趋势图、明细表、筛选器。
- SaaS 小工具：登录、项目空间、CRUD、协作记录。
- 业务流程系统：表单提交、状态流转、审核、通知。

### AI + 全栈组合

这是最适合实习求职的方向，因为它能同时展示产品、前端、后端和 AI 落地能力：

- AI 简历匹配系统。
- AI 客服质检平台。
- AI 知识库问答后台。
- AI 招聘 JD 分析器。
- AI 竞品评论洞察工具。

## 4. 压缩 MVP 范围

一个 1-3 天 demo 只保留一个主流程。

推荐结构：

- 1 个明确用户：例如 HR、运营、客服、老师、销售。
- 1 个真实输入：例如 JD、简历、客服记录、评论、文档。
- 1 个 AI 处理过程：分类、总结、问答、评分、生成。
- 1 个可视化结果：表格、卡片、报告、图表、可复制文案。
- 1 个工程亮点：检索引用、流式输出、权限、多轮记录、部署、缓存。

不要一开始就做：

- 多角色复杂权限。
- 完整支付系统。
- 大规模实时协作。
- 复杂推荐算法。
- 过多页面和配置项。

## 5. 推荐技术栈

根据“能快速交付 + 面试好解释”选择。

### 最快全栈 demo

- Frontend: Next.js 或 React + Vite
- Backend: Next.js API Routes / Express / FastAPI
- UI: Tailwind CSS / shadcn/ui / Ant Design
- Database: SQLite / Supabase / PostgreSQL
- AI: OpenAI API / 兼容 OpenAI SDK 的模型服务
- Deploy: Vercel + Supabase，或 Render/Railway

### 本地优先 demo

- Frontend: Vite React
- Backend: FastAPI
- Storage: SQLite
- AI: 可配置 API key
- Demo: 本地运行 + 录屏 + 截图

## 6. 48 小时交付节奏

### 第 0-2 小时：定位

- 读 JD，圈出 3 个最高频能力。
- 选一个业务场景。
- 写一句项目定位：这是一个给谁用、解决什么问题、输出什么结果的工具。
- 定义 3-5 个 MVP 功能。

### 第 2-8 小时：骨架

- 搭建前端页面。
- 搭建后端 API。
- 准备 mock 数据。
- 跑通主流程。

### 第 8-18 小时：核心功能

- 接入 AI 能力。
- 完成数据输入、处理、输出。
- 加入错误状态和 loading 状态。
- 保存历史记录或导出结果。

### 第 18-30 小时：产品感

- 优化页面信息层级。
- 添加示例数据。
- 做结果解释和可复制内容。
- 补充空状态、失败状态、边界提示。

### 第 30-40 小时：部署和材料

- 部署可访问版本。
- 写 README。
- 截图 3-5 张。
- 准备 90 秒演示脚本。
- 生成简历 bullet 和 HR 话术。

### 第 40-48 小时：投递使用

- 把项目和岗位 JD 对齐改写。
- 给 HR 发短消息。
- 记录反馈。
- 针对相似岗位复制改造。

## 7. 展示材料格式

每个 demo 最终至少要有：

- 项目名称。
- 一句话定位。
- 在线访问链接或本地运行说明。
- 3 张截图：输入页、处理页、结果页。
- 技术栈。
- 核心功能列表。
- 和目标岗位的对应关系。
- 简历 bullet。
- HR 开场消息。

## 8. 简历表达公式

不要写：

> 使用 React 和 Node.js 做了一个 AI 项目。

推荐写：

> 面向招聘场景独立开发 AI JD 分析与候选人匹配 demo，支持 JD 解析、能力标签提取、简历匹配评分与面试问题生成；使用 React + Node.js + LLM API 完成前后端闭环，并部署为可访问演示版本，用于针对 AI 应用实习岗位展示产品落地能力。

公式：

> 面向 [业务场景] 独立开发 [产品名称]，支持 [核心功能 1]、[核心功能 2]、[核心功能 3]；使用 [技术栈] 完成 [工程闭环]，体现 [岗位要求能力]。

## 9. HR 沟通原则

消息要短，重点是“我看了你们岗位，并做了一个相关 demo”。

结构：

1. 说明你是谁。
2. 点出岗位相关能力。
3. 放出 demo 链接或说明。
4. 请求一次沟通机会。

例子：

> 您好，我是大二学生，正在找 AI 应用/全栈方向实习。我看了岗位里对 AI 应用落地和前后端开发的要求，基于招聘场景做了一个 AI JD 分析与候选人匹配 demo，包含 JD 解析、匹配评分和面试题生成，想用这个项目更具体地展示我的开发能力。方便的话想争取一次沟通机会。

## 10. 迭代策略

不要每个岗位都从零做。建议维护 3 个底座：

- AI 文档/知识库底座：适合 RAG、客服、教育、企业知识管理岗位。
- AI 数据分析底座：适合运营、增长、BI、评论分析岗位。
- AI 招聘/销售流程底座：适合 B2B SaaS、CRM、HRTech、内部工具岗位。

每投一个岗位，改三处：

- 改业务输入。
- 改结果输出。
- 改展示文案。
