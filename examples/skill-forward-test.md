# Skill Forward Test: BOSS Job Screenshot to Three Project Routes

这个样例用于测试 `$jd-to-demo-workflow` 是否能把岗位截图/OCR 和求职者诉求转换成三条可选择的项目路线，并在用户确认后进入项目生成。

## 输入

```text
Use $jd-to-demo-workflow.

下面是一个 BOSS 直聘岗位截图 OCR 出来的内容：

岗位：AI 应用开发实习生
公司业务：企业招聘 SaaS / HRTech
职责：
1. 参与 AI 应用产品开发，负责前后端功能实现。
2. 接入大模型能力，支持知识库问答、简历解析、候选人匹配等场景。
3. 与产品和后端同学协作，优化 prompt 和用户体验。

要求：
1. 熟悉 React 或 Vue。
2. 了解 Node.js 或 Python。
3. 了解 LLM API、RAG、Prompt Engineering。
4. 有完整项目经验或可展示 demo 优先。

我的背景：
- 大二
- 正在找 AI 应用 / 全栈工程师实习
- 会 React、Node.js、Python，了解 LLM API
- 没有正式实习经历

我的诉求：
- 希望 48 小时内做出能给 HR 看的求职 demo
- 能写进简历
- 能让 Codex 在我确认路线后直接生成项目
- 不要把 API key 写进前端

请先不要直接生成项目代码。
请先给我三条路线：
1. 最快落地路线
2. 岗位命中路线
3. 技术展示路线

每条路线都要说明项目内容、产品逻辑、技术栈、MVP 范围、风险和求职表达。
最后推荐一条路线，并等待我确认。
```

## 期望输出结构

skill 的回答应该包含：

1. 岗位信息识别
2. 招聘需求理解
3. 求职者画像与 applicant situation
4. 三条项目路线
5. 路线对比和推荐
6. 等待用户确认

## 期望路线

三条路线可以类似：

- 最快落地路线：AI JD 分析工作台，优先做可演示闭环。
- 岗位命中路线：AI JD 分析与候选人匹配助手，强贴合 HRTech 招聘业务。
- 技术展示路线：带后端 API、JSON 结构化输出、mock/真实 API 双模式和报告历史的全栈 AI 工作台。

必须体现：

- 岗位 JD 解析
- 能力标签提取
- 候选人/项目材料匹配评分
- 面试题生成或评估报告
- 前端页面、后端 API、数据结构、AI prompt
- mock 模式和环境变量 API key 约束

## 不合格输出

如果 skill 只输出以下内容，就算不合格：

- 只给一个项目建议，没有三条路线。
- 直接生成完整项目，跳过用户确认。
- 只说“做一个 AI 助手”，没有页面/API/数据结构。
- 没有处理 API key 安全问题。
- 伪造实习经历。
- 项目范围大到 48 小时做不完。
- 没有 HR 话术和简历 bullet 角度。
