# Demo Types

## recruiting

Use for HRTech, recruiting, JD analysis, resume screening, candidate matching, interview workflows.

- Name: AI JD Analysis and Candidate Matching Assistant
- User: HR, recruiter, hiring ops
- Input: JD, resume, project description
- Output: skill tags, match score, gaps, interview questions
- MVP: JD parsing, candidate scoring, interview question generation, report copy/export
- API examples: `/api/analyze-job`, `/api/match-candidate`, `/api/interview-questions`

## customer-support

Use for customer service, tickets, quality inspection, operations.

- Name: AI Ticket QA and Summary Desk
- User: support lead, ops
- Input: chat logs, tickets, user feedback
- Output: summary, category, sentiment, QA score, suggestions
- MVP: ticket summary, auto category, QA scoring, improvement suggestions

## education

Use for education technology, courseware, tutoring, learning platforms.

- Name: AI Study Q&A and Mistake Explanation Assistant
- User: student, teacher, teaching ops
- Input: course content, question, student answer
- Output: explanation, key points, likely mistakes, practice plan
- MVP: material Q&A, step explanation, knowledge tags, learning advice

## operations

Use for user feedback, growth, content, e-commerce, reviews, product operations.

- Name: AI User Feedback Insight Tool
- User: ops or product team
- Input: comments, surveys, feedback records
- Output: topic clusters, sentiment, opportunities, actions
- MVP: feedback clustering, sentiment analysis, opportunity extraction, action suggestions

## fullstack-dashboard

Use when the role emphasizes CRUD, internal tools, dashboards, database, backend APIs, deployment.

- Name: AI Internship Project Management Dashboard
- User: student applying for internships
- Input: jobs, demo briefs, development status, application feedback
- Output: job board, project status, feedback metrics
- MVP: job CRUD, brief management, progress board, application feedback stats

## generic-ai-workbench

Use when the JD is broad or business context is unclear.

- Name: AI Business Process Analysis Workbench
- User: target business team
- Input: business text, table data, workflow notes
- Output: structured analysis, priority suggestions, copyable report
- MVP: input parsing, AI structured analysis, visualization, history/export
