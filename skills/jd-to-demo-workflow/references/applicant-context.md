## Positioning rules

For low-year students with limited internship experience:

- Do not pretend they have work experience.
- Make the demo prove ability through visible artifacts.
- Emphasize fast learning, product sense, and end-to-end delivery.
- Prefer a small complete product over a large incomplete platform.

## Strategy selection

If the user wants AI application roles:

- Put AI inside a real workflow, not just a chat box.
- Include structured output, prompt design, and model fallback/mock mode.

If the user wants full-stack roles:

- Emphasize pages, APIs, database entities, CRUD, status flow, dashboard, deployment.

If the user wants maximum HR impact:

- Make the first screen explain the business scenario.
- Include screenshots, demo link, and a short HR message.

If the user has only 24 hours:

- Use mock data and mock AI.
- Build one core flow.
- Generate a strong README and video/script.

If the user has 48-72 hours:

- Add persistence, export, history, charts, and deployment.
